Detailed Description

template<class Loader, class Saver>
class MetalFury::Core::Systems::MetalFurySettingsManager< Loader, Saver >

Manager for MetalFury settings supporting multiple files.

This template provides a thin wrapper around a loader and saver implementation (provided via the template parameters). It allows multiple settings files to be loaded simultaneously; each loaded file is registered in the global ConfigurationManager under the container name "MFSettings_<filenameStem>" where filenameStem is the filename without directory or extension. The manager records the original filepath for each loaded file so save operations can write back to the original location.

The loader and saver types must implement the following signatures:

  • bool loadFile(std::string filename, std::shared_ptr<ConfigContainer> container)
  • bool saveFile(const std::string& filename, std::shared_ptr<ConfigContainer> container)

All public APIs that operate on a specific file require the filename id (the filename stem) rather than a full path. This simplifies callers when multiple files are in use.

Template Parameters
LoaderType implementing IMF_ConfigurationFileLoader semantics.
SaverType implementing IMF_ConfigurationFileSaver semantics.

template<class Loader , class Saver >
bool MetalFury::Core::Systems::MetalFurySettingsManager< Loader, Saver >::loadSettings ( const std::string & filepath)
inline

Load settings from a file and register a container for it.

Validates that the provided filepath has a ".mfsettings" extension (case-insensitive). If validation fails the method logs a warning and returns false. The method invokes the Loader::loadFile to populate a ConfigContainer; if the loader returns false the method logs a warning and returns false. On success a metadata element named "__mf_filepath" is added to the container containing the original filepath, the container is registered under the name "MFSettings_<stem>", and the original filepath is recorded for future saves.

Parameters
filepathFull path to the settings file to load.
Returns
true on successful load and registration; false on validation or loader failure.
